# Приватность и обработка данных

Практические правила для разработки и модерации. Не заменяют консультацию юриста перед запуском.

## Принцип минимизации данных

Собираем только то, что нужно для:

- приёма и модерации отзыва;
- связи с автором через Telegram;
- разрешения споров;
- защиты сервиса от злоупотреблений.

Всё лишнее не запрашиваем. Регистрация на сайте в MVP не требуется.

## Что где хранится

| Данные | Хранение | Доступ |
|--------|----------|--------|
| Публичный текст отзыва | Supabase PostgreSQL | Все посетители сайта |
| Исходный текст до очистки | Supabase PostgreSQL | Модератор, автор |
| Telegram user ID | Supabase PostgreSQL | Сервис (бот, модерация) |
| Доказательства (фото, PDF) | Supabase Storage (private bucket) | Модератор, автор; не в браузере публики |
| Логи модерации | Supabase PostgreSQL | Только модераторы |
| Жалобы и материалы `disputed` | Supabase PostgreSQL + Storage | Только модераторы |

## Публичный текст

Перед публикацией модератор (или автоматический препроцессор на следующих этапах) **очищает** текст от:

- ЕГН, ЛНЧ, паспортных данных;
- телефонов, email, ссылок на личные профили;
- полного адреса частного лица;
- ФИО, если они позволяют идентифицировать человека вне публичного контекста организации.

На сайте показывается **только очищенная версия**. Исходник не доступен публично.

## Доказательства — только приватно

Пользователь может приложить договор, переписку, фото повреждений как обоснование отзыва или спора.

Правила:

- Файлы лежат в **приватном** Storage bucket (`review-attachments`).
- В публичную часть не попадают.
- Модератор просматривает их для проверки фактов.
- Персональные документы (паспорт, удостоверение) **не публикуются** и по возможности не запрашиваются; если приложены — не использовать в публичном тексте.

## Персональные документы

- Не просим загружать паспорт или ЕГН для публикации отзыва.
- Если пользователь приложил лишнее — модератор не переносит данные в публичный текст.
- Документы с ПДн хранятся минимально необходимое время для разрешения спора.

## Права пользователя

Пользователь (автор отзыва) может через бота или обращение к модерации:

- **запросить исправление** опубликованного текста;
- **запросить удаление** своего отзыва и связанных приватных данных;
- получить информацию о статусе модерации.

Сроки обработки запросов — в разумные сроки (целевое значение для MVP: до 14 дней).

Удаление:

- публичный отзыв снимается (`removed` или полное удаление записи);
- приватные вложения удаляются из Storage;
- логи модерации могут сохраняться обезличенно для защиты сервиса (см. ниже).

## Логи модерации

Храним записи вида: `review_id`, `moderator_id`, `action`, `timestamp`, `comment`.

Зачем:

- разбор споров и жалоб;
- защита от необоснованных претензий к сервису;
- аудит действий модераторов.

Логи **не публикуются**. Доступ — только администраторам.

## Ключи и доступ

| Секрет | Где использовать |
|--------|------------------|
| `SUPABASE_SERVICE_ROLE_KEY` | Бот, серверные API — **никогда в браузере** |
| `SUPABASE_ANON_KEY` / `NEXT_PUBLIC_*` | Сайт — только операции, разрешённые RLS |
| `TELEGRAM_BOT_TOKEN` | Только сервер/бот, в `.env` |

RLS в Supabase: анонимный пользователь сайта читает только `approved` отзывы без приватных полей.

## Поиск

Публичный поиск — по объекту, городу, названию агентства или УК.

**Поиск по ФИО физических лиц не предусмотрен** и не должен добавляться без пересмотра этих правил.
