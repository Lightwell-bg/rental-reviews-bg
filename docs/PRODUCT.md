# Продукт: Rental Reviews BG

## Что это за площадка

Rental Reviews BG — сервис проверенных отзывов об опыте аренды недвижимости в Болгарии. Пользователи читают отзывы на сайте и подают свои через Telegram-бот. Каждый отзыв проходит ручную модерацию до публикации.

Цель — дать реальную картину по объектам, арендодателям, арендаторам, агентствам и управляющим компаниям, не превращаясь в доску объявлений и не раскрывая личные данные участников.

## Кто пользователи

| Роль | Что делает |
|------|------------|
| **Арендатор** | Ищет жильё, читает отзывы, оставляет свой опыт аренды |
| **Арендодатель** | Сдаёт жильё, читает отзывы, делится опытом сдачи |
| **Агентство / риелтор** | Участвует в сделках — на него могут оставить отзыв или ответить на отзыв |
| **Управляющая компания** | Обслуживает дом/комплекс — объект отзывов и ответов |
| **Модератор** | Проверяет отзывы, запрашивает правки, разрешает споры |
| **Посетитель сайта** | Читает опубликованные отзывы без регистрации |

Публичный поиск — **по объекту, городу, названию агентства или УК**, но **не по ФИО** физических лиц.

## Сценарии MVP

### 1. Просмотр отзывов (сайт)

1. Пользователь открывает сайт.
2. Ищет по адресу, городу, названию агентства или УК.
3. Видит карточку объекта/субъекта и список отзывов со статусом `approved`.
4. Может прочитать официальный ответ второй стороны (если есть).

### 2. Подача отзыва (Telegram-бот)

1. `/start` → выбор типа отзыва.
2. Указание объекта: адрес, ссылка на агентство/УК (в зависимости от типа).
3. Текст отзыва + опционально фото/документы как **доказательства** (хранятся приватно).
4. Отзыв сохраняется со статусом `pending`, модератор получает уведомление.

### 3. Модерация (админ)

1. Модератор видит очередь `pending`.
2. Проверяет текст, очищает персональные данные, при необходимости запрашивает правки (`request_changes`).
3. Публикует (`approve`) или отклоняет (`reject`).
4. При жалобе второй стороны — статус `disputed`, разбор и решение.

### 4. Право ответа (вторая сторона)

1. Уполномоченный представитель (арендодатель, агентство, УК) получает уведомление о публикации отзыва.
2. Может подать **официальный ответ** через бота или админку.
3. Ответ тоже проходит модерацию и публикуется рядом с отзывом.

## Что публикуется публично

- Тип и тема отзыва (`property`, `landlord`, `tenant`, `agency`, `management_company`).
- Очищенный текст отзыва (без персональных данных).
- Привязка к объекту: город, район, улица/ж.к., дом/блок (или `X`), квартира — опционально; после одобрения квартира видна на сайте.
- Оценка / теги (если предусмотрены в MVP).
- Дата публикации.
- Официальный ответ второй стороны (после модерации).
- Агрегированная статистика (количество отзывов, средняя оценка).

## Что хранится только приватно

- Telegram user ID и внутренний ID автора отзыва.
- Исходный текст до очистки модератором.
- Загруженные доказательства (договор, переписка, фото) — в Supabase Storage, доступ только модераторам и автору.
- Контактные данные для связи с модерацией.
- Логи модерации: кто, когда, какое действие (`approve`, `reject`, `request_changes`, `remove`).
- Жалобы и материалы по статусу `disputed`.
- Служебные заметки модератора.

## Типы отзывов

| Код | О чём | Кто обычно пишет | Привязка |
|-----|-------|------------------|----------|
| `property` | Состояние жилья, инфраструктура, соседи, коммунальные платежи | Арендатор | Адрес объекта (без точной квартиры в публичной части) |
| `landlord` | Взаимодействие с арендодателем: депозит, ремонт, договор | Арендатор | Объект + обезличенный профиль арендодателя |
| `tenant` | Опыт сдачи: состояние жилья после аренды, соблюдение договора | Арендодатель | Объект, **без идентификации** арендатора |
| `agency` | Работа риелторского агентства: комиссии, прозрачность, сервис | Арендатор или арендодатель | Название агентства |
| `management_company` | Управляющая компания: содержание дома, реакция на заявки | Арендатор или собственник | Название УК + адрес здания |

### Правила по типам

- **`property`** — фокус на объекте, не на личности.
- **`landlord`** / **`tenant`** — описание фактов и опыта, без ФИО и контактов второй стороны.
- **`agency`** / **`management_company`** — можно указывать название организации; персональные данные сотрудников (телефоны, email) в публичный текст не попадают.

## Статусы отзыва (кратко)

| Статус | Значение |
|--------|----------|
| `pending` | На модерации |
| `approved` | Опубликован |
| `rejected` | Отклонён |
| `request_changes` | Нужна правка от автора |
| `disputed` | Спор, разбирает модератор |
| `removed` | Снят с публикации |

## Ограничения продукта

- Нет публичного поиска по ФИО.
- Нет публикации ЕГН, ЛНЧ, паспортных данных, телефонов, email.
- Нет автоматической публикации — только после ручной проверки.
