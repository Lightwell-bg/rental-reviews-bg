# Модерация отзывов

## Общий принцип

**Все отзывы и официальные ответы проходят ручную проверку** до публикации. Автоматическая публикация не используется.

Модераторы — пользователи из `ADMIN_TELEGRAM_IDS` и будущая web-админка `/admin`.

## Статусы

```
pending → approve → approved (публично)
        → reject  → rejected
        → request_changes → (автор правит) → pending
        → disputed (при жалобе второй стороны)
approved → remove → removed
disputed → approve | reject | remove
```

| Статус | Описание |
|--------|----------|
| `pending` | Новый отзыв в очереди |
| `approved` | Опубликован на сайте |
| `rejected` | Не публикуется, автор может получить причину |
| `request_changes` | Отправлен автору на доработку |
| `disputed` | Спорная ситуация, разбор с учётом позиции второй стороны |
| `removed` | Был опубликован, снят с сайта |

## Действия админа

| Действие | Когда применять |
|----------|-----------------|
| `approve` | Текст очищен, факты изложены корректно, нет запрещённого контента |
| `reject` | Отзыв не соответствует правилам, исправить невозможно |
| `request_changes` | Отзыв можно опубликовать после правок (убрать ПДн, смягчить формулировки) |
| `remove` | Уже опубликованный отзыв нужно снять (жалоба, ошибка, устаревание) |

Каждое действие фиксируется в **логе модерации** (кто, когда, комментарий).

## Запрещённый контент

### Персональные данные

В публичный текст **нельзя** включать:

- **ЕГН** (болгарский идентификатор)
- **ЛНЧ** (личен номер на чужденец)
- Паспортные данные, номера документов
- Телефоны, email, ники в мессенджерах
- **Полный домашний адрес частного лица** (допустим адрес объекта аренды в обезличенном виде: город, улица, дом — без квартиры; для частных лиц — без точного адреса проживания)

Модератор вычищает такие данные из публикуемой версии. Исходник и доказательства остаются в приватном хранилище.

### Обвинения и тон

- **Прямые обвинения в преступлениях или мошенничестве без доказательств** — отклонить или перевести в фактологическое описание («депозит не вернули в течение 60 дней» вместо «мошенник»).
- **Угрозы** — немедленный `reject`, при необходимости блокировка автора.
- **Оскорбления** — `request_changes` или `reject`.
- **Дискриминация** (национальность, пол, религия и т.д.) — `reject`.

### Содержание отзыва

Отзыв должен описывать **факты и личный опыт аренды**, а не:

- сплетни и домыслы;
- рекламу конкурирующих услуг;
- отзывы не по теме (политика, личные конфликты вне аренды).

## Право ответа второй стороны

1. После публикации отзыва типов `landlord`, `tenant`, `agency`, `management_company` вторая сторона может подать **официальный ответ**.
2. Ответ проходит ту же ручную модерацию.
3. В ответе действуют те же запреты на ПДн, угрозы и оскорбления.
4. Ответ публикуется под отзывом с пометкой «Официальный ответ».

Если вторая сторона не согласна с отзывом:

1. Подаёт жалобу через бота или админку.
2. Отзыв переводится в статус **`disputed`**.
3. Модератор запрашивает доказательства у обеих сторон (хранятся приватно).
4. Принимает решение: оставить (`approve`), снять (`remove`), отклонить изначальный отзыв (`reject`) или запросить правки.

## Чеклист модератора

1. Тип отзыва соответствует содержанию?
2. Привязка к объекту/организации корректна?
3. Нет ЕГН, ЛНЧ, паспортов, телефонов, email, полного адреса частного лица?
4. Текст фактологичен, без необоснованных обвинений?
5. Нет угроз, оскорблений, дискриминации?
6. Доказательства (если есть) просмотрены, в публикацию не попадают?
7. Для `tenant` — арендатор не идентифицируется?
8. Действие и комментарий записаны в лог?

## Уведомления

| Событие | Кому | Как |
|---------|------|-----|
| Новый `pending` | модераторам | Telegram (`/admin`) |
| `request_changes` | автору | Telegram + «Мои заявки» в боте |
| `rejected` | автору | Telegram + комментарий в карточке заявки |
| `approved` | автору | Telegram (опционально) |
| `disputed` | модераторам | (планируется) |

### Request changes — что видит автор

1. **Push в Telegram** — сразу после действия модератора (из бота или web `/admin`).
2. Текст включает статус, ID заявки и **комментарий модератора** (`moderation_notes`).
3. Если модератор не оставил комментарий (модерация из Telegram), подставляются подсказки из `ai_flags.moderation_notes`.
4. Кнопки «Открыть заявку» / «Мои заявки» ведут в бота.
5. В карточке заявки (`Мои заявки` → заявка) тот же комментарий виден постоянно.
6. Кнопка **«Исправить и отправить снова»** — автор правит заголовок, текст, приватный комментарий; статус → `pending`, модераторы получают уведомление.

**Цикл доработки:** `pending` → `request_changes` → (автор правит в боте) → `pending` → `approved` / `reject` / снова `request_changes`.

**Web `/admin`:** при «Request changes» или «Reject» комментарий обязателен — он сохраняется в `moderation_notes` и уходит автору.

**Переменные:** для уведомлений из web добавьте `TELEGRAM_BOT_TOKEN` в `web/.env.local` (тот же токен, что у бота).
