# Промпты для поэтапной разработки

Шаблоны запросов для Cursor. Копируйте и адаптируйте под текущий этап.

## Этап 1: Схема базы данных

```
Создай SQL-миграцию в supabase/migrations/ для rental-reviews-bg:
- таблица properties (адрес, город, нормализованный ключ поиска)
- таблица reviews (тип tenant_review | landlord_review, текст, статус модерации, property_id, telegram_user_id)
- индексы для поиска по адресу, не по ФИО
Добавь RLS-политики в тот же файл миграции или в `supabase/migrations/00N_...sql`
```

## Этап 2: Сценарий бота — подача отзыва

```
Реализуй в bot/ сценарий подачи отзыва для арендатора и арендодателя:
- FSM aiogram 3
- выбор роли, адрес, текст, опционально фото
- сохранение в Supabase со статусом pending
- без публичных персональных данных
```

## Этап 3: Сайт — просмотр отзывов

```
Добавь в web/ страницу поиска отзывов по адресу/городу.
Показывай только approved отзывы.
Поиск только по объекту, не по ФИО.
```

## Этап 4: Модерация

```
Реализуй модерацию:
- уведомление админам в Telegram при новом отзыве
- web-админка /admin для approve/reject
- усиленные правила для landlord_review
```

## Этап 5: Деплой

```
Помоги задеплоить rental-reviews-bg:
- web на Vercel
- bot через docker compose на VPS
- проверь env и Supabase Storage bucket review-attachments
```
